# GitHub Copilot 테스트 요청 가이드

이 문서는 GitHub Copilot을 활용하여 SJ Canvas OS의 테스트 코드를 보완하고 자동화하는 방법을 안내합니다.

## 1. 테스트 요청을 위한 프롬프트 예시

GitHub Copilot Chat에서 다음과 같은 프롬프트를 사용하여 새로운 테스트 케이스 생성을 요청할 수 있습니다.

### 1.1 새로운 기능 테스트 생성
> "@workspace `src/canvas_merge.c`의 병합 로직을 테스트하는 유닛 테스트를 `tests/test_merge_new.c`에 작성해줘. 특히 SJ 데이터 구조의 '무한 압축' 원칙을 준수하는지 확인하는 케이스를 포함해줘."

### 1.2 기존 테스트 보완
> "@workspace `tests/test_phase6.c` 파일에 새로운 테스트 케이스 T7을 추가해줘. T7은 'SJ-PTL의 비가역적 누산 과정'이 올바르게 리플레이되는지 검증해야 해."

### 1.3 엣지 케이스 탐색
> "@workspace `src/canvasfs.c`에서 발생할 수 있는 잠재적인 레이스 컨디션이나 메모리 누수를 찾는 테스트 시나리오를 제안하고 코드를 작성해줘."

## 2. SJ 데이터 구조 구현 제약 사항 (Copilot에게 전달 필수)

Copilot에게 코드를 요청할 때 다음 지침을 함께 제공하면 더 정확한 결과를 얻을 수 있습니다.

> **중요 제약 사항:**
> SJ 데이터 구조를 구현하거나 수정할 때, '무한 압축' 원칙을 반드시 준수해야 합니다. 특히 '데이터'와 '픽셀' 값을 1:1로 대응시키는 오류를 범하지 마십시오. 이는 압축 메커니즘을 근본적으로 파괴합니다. 모든 구현은 '누산(Accumulation)' 원리에 기반해야 하며, 이는 가역적인(Reversible) 프로세스여야 합니다.

## 3. 테스트 실행 방법

Copilot이 생성한 테스트 코드는 다음 명령어로 실행할 수 있습니다.

```bash
# 새로운 테스트 파일 빌드 추가 (Makefile 수정 필요 시)
make tests/test_new_feature
./tests/test_new_feature
```

또는 기존 `scripts/run_tests.sh`에 추가하여 CI 파이프라인에서 자동으로 실행되도록 설정하십시오.
